# Skill 00 — Project Framing

## 目的
在需求不完整時，快速產出可執行範圍，避免「AI 直接開寫但方向錯誤」。

## 輸入
- 商業目標
- 使用者角色
- 時程/資源限制

## 步驟
1. 用一句話定義問題與成功條件。
2. 列出 in-scope / out-of-scope。
3. 定義 MVP：保留關鍵價值，刪掉次要能力。
4. 風險盤點：技術、資料、流程、依賴。
5. 產出 1 週內可交付里程碑。

## 輸出格式
- Problem Statement
- Success Metrics（最多 3 個）
- Scope 表格
- Milestones（D1/D3/D5）
- Top 3 Risks + Mitigation

## 反模式
- 還沒定義資料來源就設計完整架構
- 一開始就設計過多抽象層
- 把「想做的」當成「這版必做」
